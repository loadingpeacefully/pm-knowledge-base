# Worksheet Question Validation Flow

## Overview

Automated validation rules and flow for math worksheet questions generated by geeta-AI. Validates tag format, blank-key correspondence, answer type checking, and option structure before questions proceed to human review.

## Problem Statement

AI-generated questions frequently contain structural errors: wrong tag formats, mismatched blank-key counts, invalid answer types, or mixed incompatible tags. Manual detection of these errors is slow and inconsistent. An automated validator catches structural errors before human review.

## Goals & Success Metrics

- Catch > 95% of structural errors before questions reach human review
- Reduce reviewer time per question by 30% (reviewer focuses on content, not format)
- Achieve 0% formatting errors in questions that pass validation (Final Review = True)
- Validation runs complete within 5 seconds per batch of 20 questions

## User Stories / Jobs-to-Be-Done

- As a developer, I want the validator to automatically flag wrong tags so I can fix them quickly
- As a reviewer, I want to only review questions that pass structural validation
- As a content lead, I want to know exactly why a question failed validation (not just pass/fail)
- As a developer, I want to upload only validated questions to staging without manual checking

## Feature Scope

**In Scope:**
- Sheet structure validation: required columns exist (Problem Statement, Expected Answer, Final Review, etc.)
- Tag format validation: options tags ([options:single], [options:multi]), blank tags (blankN, blankN:type)
- Mutual exclusivity: blank tags and options tags cannot coexist in same question
- Blank-key correspondence: each blankN has matching keyN; counts must match exactly
- Answer type validation: numeric types accept -?\\d*\\.?\\d+ pattern; text types accept [A-Za-z]+ only
- Single option questions: exactly 1 key, value must be A-D
- Multi option questions: at least 1 key, each value single alphabetic character
- Output columns: Final Review (True/False), Different Type Input (Yes/empty), Special Symbol In Key (Yes/empty)
- Is Field Missing column: array formula checking all required fields present

**Out of Scope:**
- Content quality validation (semantic correctness of questions)
- Image validation (separate image review workflow)
- Student-facing validation

## Key Design Decisions

- Validator runs as Google Apps Script (validateProblemStatementsAndAnswers) on question sheet
- Final Review = True only when ALL validations pass — one failure = False
- Different Type Input = Yes if any :text, :fraction, :/fraction, or options tags present
- Special Symbol In Key = Yes if key values contain unexpected characters for their input type
- Validation is idempotent — can be re-run multiple times on same sheet

## Open Questions / Risks

- Should validator also check for duplicate QIDs within the same skill code?
- Risk: Apps Script execution limits may cause timeout on very large sheets
- Dependency: Question sheet must follow exact column naming convention for validator to work
- Open: How to handle questions where validator produces a False but human reviewer overrides?
