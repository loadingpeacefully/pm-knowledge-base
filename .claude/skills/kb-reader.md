# Skill: KB Reader

Use this skill whenever reading KB source files to inform lesson writing.

## Rules
- Never modify KB files (technical-architecture/, product-prd/, research-competitive/, performance-reviews/)
- Read _REGISTRY.md first to find relevant files by topic
- When a KB file mentions a company-specific name (BrightChamps product names), abstract it for the lesson (e.g. "a live class platform" not "BrightChamps")
- Extract patterns, not specifics — the lesson should generalize, the KB evidence grounds it

## Abstraction map

| Paathshala / Eklavya / Doordarshan / Chowkidar | "microservice" or name the function (curriculum service, class service) |
